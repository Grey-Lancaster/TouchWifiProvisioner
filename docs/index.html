TouchWifiProvisioner

On-device Wi-Fi provisioning for LVGL touchscreens. Scan, pick a network, and enter a password with the on-screen keyboard — no captive portal, no phone required.

View on GitHub Installation

What it looks like

A built-in dark theme for the picker, and a plain green confirmation once the device is online.

Illustrations of the actual on-device UI, not photographs of a screen.

Why it's different

Built for devices that already have their own screen.

🖐️

No captive portal

No hotspot, no borrowing a phone's browser. The device shows its own picker and keyboard directly on-screen.

🧩

LVGL-only

Only ever touches LVGL objects and WiFi/Preferences APIs — never the display bus or touch controller. Works on any panel LVGL can drive.

📐

Any resolution

Layout uses flex sizing (lv_pct) instead of hardcoded pixel coordinates, so it adapts to whatever screen the host provides.

🔌

Auto-reconnect

Saved credentials are tried first on every boot. On failure — or none saved yet — it just re-shows the picker.

Install

Arduino IDE or PlatformIO — usage code is identical either way.

lib_deps =
    https://github.com/Grey-Lancaster/TouchWifiProvisioner.git
    lvgl/lvgl@^8.3.11
#include <TouchWifiProvisioner.h>

void onWifiConnected(const String &ip) {
  Serial.printf("Connected, IP: %s\n", ip.c_str());
}

void setup() {
  // ...init display/touch/LVGL first...
  TouchWifiProvisioner::begin(lv_scr_act(), "Gramps", onWifiConnected);
}

void loop() {
  lv_timer_handler();
  TouchWifiProvisioner::loop();
}

Full setup steps (Arduino IDE vs. PlatformIO, lv_conf.h requirements) are in the README.
